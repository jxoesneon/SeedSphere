# Contributing to SeedSphere

Welcome to the Federated Frontier! We're excited to have you contribute to SeedSphere.

## üöÄ How to Contribute

### 1. Find an Issue
- Browse our [issue tracker](https://github.com/seedsphere/seedsphere/issues) for "good first issues".
- If you'd like to implement a new feature, please open a [feature request](https://github.com/seedsphere/seedsphere/issues/new?template=feature_request.md) first to discuss it.

### 2. Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Ensure your code follows the existing style guidelines.
3. Run `flutter analyze` to check for lint issues.
4. Run `flutter test` to ensure all tests pass.
5. Update documentation if necessary.
6. Open a Pull Request with a clear description of your changes.

### 3. Code Style
- We follow the official [Dart Style Guide](https://dart.dev/guides/language/effective-dart/style).
- All library files MUST include enterprise-grade documentation headers.
- Use `dart format .` before committing.
